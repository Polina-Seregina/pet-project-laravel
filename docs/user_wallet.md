### **Кошелёк пользователя**

*Кошелёк пользователя создается автоматически при регистрации пользователя.*
*При успешной регистрации генерируется событие 'Registered'. Слушатель события 'CreateUserWallet', зарегистрированный в EventServiceProvider, создает кошелёк пользователя с нулевым балансом.*
*У пользователя может существовать только один кошелёк.*


### **Описание роутов управления кошельком**

#### 1. GET /wallet
*Возвращает cтраницу кошелька авторизованного пользователя.*

*Входные данные*: Request
*Формат ответа*: (200 OK) - text/html

#### 2. PATCH /wallet
*Обновляет данные в кошельке пользователя - увеличивает баланс на указанную сумму.*

*Входные данные*: WalletTopUpRequest

| Параметр | Тип | Обязательный | Правила валидации | Описание |
| -------- | ----|--------------|-------------------|----------|
| 'amount' | numeric | Да | required, numeric, min:0| Сумма пополнения кошелька |

*Формат ответа*: (302 OK)

#### 3. GET /wallet/replenishment
*Возвращает форму для пополнения кошелька пользователя.*

*Входные данные*: Request
*Формат ответа*: (200 OK) - text/html
